## Breaking changes

**`FlatKVCache` mutators now return `Result` instead of panicking** (#290, PR #298).
`append_kv` changed from `usize` to `Result<usize>`; `advance`, `advance_by`, and
`append_kv_all_layers` changed from `()` to `Result<()>`. Callers must propagate with `?`
or match on the error.

Migration:

```rust
// before (v0.3.0)
let n = cache.append_kv(layer, k, v);

// after (v0.4.0)
let n = cache.append_kv(layer, k, v)?;
```

No other public API changed shape.

## New (additive)

**Inference — streaming generation.** `generate_streaming` with UTF-8-safe detokenization
flushes only complete codepoints per token. `chat_metal` exposes streaming, sampler controls
(temperature, top-k, repetition penalty), and LoRA adapter loading.

**Inference — performance kernels.** Tiled simdgroup Q4 GEMM enabled on Apple7+ (1.62x
prefill, PR #265); Q8 tiled simdgroup GEMM for Q8 prefill (#283); half-tile staging
reduces threadgroup memory 16 → 10 KB (#270); float4 X loads + unrolled MMA inner loop
(#274); chunked-parallel GDN prefill scan, default-on (~1.6x prefill, #235); batched
causal prefill attention (3.03x chunk attention, 5.31x cumulative TTFT, #229); chunked
batched prefill for long prompts (1.78x TTFT, #228); fused RMSNorm+residual batch
dispatches in Metal prefill path.

**Inference — f16 KV cache (opt-in).** Metal GQA path accepts f16 KV tensors via
feature flag, halving GPU KV memory footprint (#238).

**Inference — KV cache capacity cap.** Opt-in hard cap on KV allocation with
below-prompt guard (#215).

**Inference — GPU profiler.** Env-gated (`LATTICE_GPU_PROFILE`) 4-category decode
profiler, off by default (#241).

**Inference — perplexity + QuaRot CLI.** Perplexity CLI emits line-oriented JSON
events for Studio integration (#200). QuaRot dry-run emits byte-accounting events (#201).

**Tune — `train-backward` feature gate.** Exact-gradient LoRA backward pass with
multi-layer training pipeline, feature-gated as `train-backward` (#191). `generate_lora`
binary adds streaming output and sampler controls.

**Embed — `embed` CLI.** Download-only mode and JSON event output (#199).

**Bench — decode slope harness.** ADR-064 Phase-0: decode slope/intercept fit with
bootstrap CI (#234).

**CI — e2e-parity gate.** Replaced the flaky Criterion bench-regression gate with a
same-runner HF transformers vs lattice greedy-token agreement check. First 3 tokens must
match HF exactly (#192). Hardened merge gate: feature-matrix, bench-compile, cargo-deny,
parity passthrough (#204). App-shipped binaries are compile-gated in CI (#203).

## Known limitations

Grammar-constrained generation (`pub mod grammar`, opt-in via `GenerateConfig::grammar`)
has two pre-existing limitations present since v0.3.0:

- **Object schemas with mixed required+optional properties may reject valid JSON** (#355).
  The PDA state machine can refuse a well-formed object that satisfies such a schema.

- **The PDA backtracker does not rewind consumed bytes** (#353). Some shared-prefix
  `anyOf`/`oneOf` grammars over-accept or over-reject at branch points requiring
  lookahead beyond a committed prefix.

Both are tracked. Greedy generation without grammar constraints is unaffected.
